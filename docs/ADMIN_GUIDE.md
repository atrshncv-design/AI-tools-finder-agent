# Руководство администратора

## Доступ к панели

Откройте `/admin`. Первый пользователь с `OWNER_UNION_ID` получает роль `admin` автоматически. Других администраторов можно назначить в разделе "Пользователи".

## Управление пользователями

- В разделе **Пользователи** отображаются все зарегистрированные пользователи.
- Нажмите **Сделать админом** / **Снять админа** для изменения роли.
- Роль `admin` требуется для доступа к панели и ручного запуска агентов.

## Управление источниками

### Типы источников

- **RSS** — лента в формате RSS/Atom. В config хранится `feedUrl`.
- **HTML** — парсинг веб-страницы с CSS-селекторами. В config хранится `selector`.
- **Google News** — RSS-лента Google News. В config хранится `feedUrl`.
- **API** — JSON API. В config хранятся пути к полям (`articlesPath`, `titlePath`, `urlPath` и т.д.).

### Добавление источника

1. Введите название и URL.
2. Выберите тип.
3. Нажмите **Добавить**.

### Отключение / удаление

- Переключатель **Активен** отключает источник без удаления.
- Иконка корзины удаляет источник и связанные логи.

## Ручной запуск

- **Запустить парсинг** — обходит все активные источники и добавляет новые статьи.
- **Суммаризовать** — обрабатывает до 10 непрочитанных/несуммаризованных статей.

## Мониторинг

- **LM Studio** — статус подключения к локальной модели.
- **Агенты** — текущий статус parse-agent и summarize-agent.
- **Health источников** — цветной индикатор рядом с каждым источником.
- **Логи парсинга** — результаты последних проходов по источникам.

## Расписание

Парсинг запускается автоматически:

- 06:00 — основной ежедневный парсинг
- 07:00 — парсинг научных инструментов
- каждые 4 часа — дополнительная проверка новостей

## Перевод через GigaChat

Полный перевод статей выполняется через GigaChat API (`GIGACHAT_API_KEY`, `GIGACHAT_MODEL`). LLM используется для саммари и/или перевода в зависимости от SUMMARY_PROVIDER и TRANSLATION_PROVIDER.

## Подключение LLM

1. Установите и запустите LM Studio.
2. Загрузите модель `google/gemma-4-e4b` (или аналогичную, совместимую с OpenAI API).
3. Измените `LM_STUDIO_MODEL` в `.env`.
4. Перезапустите контейнер `app`.

Параметры `LM_STUDIO_TIMEOUT_MS`, `LM_STUDIO_MAX_INPUT_TOKENS`, `LM_STUDIO_CONCURRENCY` можно подстроить под возможности железа.
