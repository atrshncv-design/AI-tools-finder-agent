# Пользовательское руководство

## ИИ-новостной агент для научного журнала

Добро пожаловать в систему мониторинга новостей искусственного интеллекта! Это руководство поможет вам эффективно использовать все возможности сервиса.

---

## Авторизация

### Вход в систему

1. Нажмите кнопку **"Войти"** в правом верхнем углу
2. Вы будете перенаправлены на страницу авторизации Kimi Platform
3. Подтвердите вход
4. После успешной авторизации вы вернётесь на главную страницу

### Роли пользователей

| Роль | Возможности |
|---|---|
| **User** | Просмотр новостей, поиск, избранное, прочтение |
| **Admin** | Все возможности User + управление парсингом и суммаризацией |

---

## Работа с лентой новостей

### Просмотр новостей

- На главной странице отображаются все новости, отсортированные по дате
- Непрочитанные новости выделены цветом (синяя полоса слева)
- Badge-счётчик в хедере показывает количество непрочитанных

### Фильтрация

Используйте горизонтальную панель категорий для фильтрации:

- **Все** — показать все новости
- **Новая LLM** — новые языковые модели
- **ИИ-агент** — агентные системы
- **Сравнение** — сравнения моделей
- **Бенчмарки** — тестирования и оценки
- **Обновления** — обновления существующих систем

### Чтение статьи

1. Нажмите на заголовок или кнопку **"Подробнее"**
2. Откроется страница со всей информацией:
   - Краткое саммари (3-5 предложений)
   - Подробный перевод (10-15 предложений)
   - Метаданные (источник, дата, теги)
3. Ставь автоматически отмечается как прочитанная

### Действия со статьёй

| Кнопка | Описание |
|---|---|
| **В избранное** | Сохранить статью в закладки |
| **Читать оригинал** | Перейти к оригинальной статье |
| **Полный перевод** | Получить полный перевод на русский язык |

## Избранное

### Добавление в избранное

1. Откройте статью
2. Нажмите кнопку **"В избранное"** (иконка звезды)
3. Кнопка изменится на **"В избранном"**

### Просмотр избранного

1. Перейдите в раздел **"Избранное"** через хедер
2. Отображаются все сохранённые статьи
3. Можно удалить статью из избранного

### Счётчик избранного

Badge-счётчик в хедере показывает количество сохранённых статей.

## Поиск

### Поиск по новостям

1. Введите поисковый запрос в поле поиска в хедере
2. Нажмите Enter или кнопку поиска
3. Отобразятся результаты, содержащие запрос в заголовках и аннотациях

### Примеры запросов

- `GPT-5` — найти новости о GPT-5
- `нейросеть` — все новости о нейросетях
- `AlphaFold` — новости о AlphaFold

## Панель управления (Admin)

Доступна только для пользователей с ролью **Admin**.

### Статус системы

Отображает статус подключения к LM Studio:
- Зелёный кружок — LM Studio подключён
- Красный кружок — LM Studio недоступен

### Действия

| Кнопка | Описание |
|---|---|
| **Запустить парсинг** | Запустить сбор новостей из всех источников |
| **Суммаризовать** | Запустить генерацию аннотаций для новых статей |

### Источники

Отображает список всех источников новостей и их статус:
- **Активен** — источник включён и парсится
- **Отключён** — источник временно отключён

### Логи парсинга

Показывает последние операции парсинга:
- Статус (completed/failed/running)
- Количество найденных и новых статей
- Сообщения об ошибках

## Тема оформления

### Переключение темы

Нажмите иконку солнца/луны в хедере для переключения между:
- **Светлая тема** — для работы днём
- **Тёмная тема** — для работы в тёмное время суток

Настройки темы сохраняются в браузере.

## Часто задаваемые вопросы

### Почему статья не отображается в ленте?

Возможные причины:
1. Источник временно недоступен
2. Статья не прошла фильтр по ключевым словам
3. Ст уже есть в базе (дедупликация)

### Почему суммаризация не работает?

Проверьте:
1. LM Studio запущен и доступен на `localhost:1234`
2. Модель загружена в LM Studio
3. В настройках системы LM Studio показывает статус "подключён"

### Как добавить новый источник?

Это задача для администратора:
1. Добавьте источник в базу данных через API
2. Укажите URL, тип (RSS/HTML) и селектор
3. Включите источник

### Как изменить расписание парсинга?

Отредактируйте cron-выражения в файле `api/scheduler/index.ts`:

```typescript
const PARSE_CRON = "0 6 * * *";      // 06:00 ежедневно
const SUMMARIZE_CRON = "30 6 * * *";  // 06:30 ежедневно
const HOURLY_CRON = "0 */4 * * *";    // каждые 4 часа
```
